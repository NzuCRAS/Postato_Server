# 内部效能平台 – 整体构建设计

## 一、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | React 18 + TypeScript | 使用 Vite 构建，组件库可选 Ant Design 或 MUI（快速搭建后台页面） |
| **后端** | Java 17 + Spring Boot 3 | 成熟稳定，一人开发推荐 Spring Initializr 快速启动 |
| **数据库** | MongoDB 7（云服务或自建） | 文档模型适配树状结构、嵌套文档、可变字段 |
| **MCP Server** | TypeScript + `@modelcontextprotocol/sdk` | 独立 Node 服务，协议层 |
| **部署** | 单台云服务器（如阿里云/腾讯云 2C4G）+ Nginx + Docker Compose | 前端静态文件、后端 jar、MCP Server 均容器化 |
| **认证** | JWT（Web） + API Key（MCP） | 自建用户表，独立身份认证 |
| **AI 能力** | 可集成 OpenAI/Claude API，也可仅在 MCP 侧通过 Claude Code 使用 | 平台内部 Agent 可后续扩展 |

## 二、项目整体架构

```
用户浏览器
    │
    ├── Web 前端 (React SPA) 端口 80/443
    │       ├── 需求管理
    │       ├── 开发进度树
    │       ├── 知识库 Wiki
    │       └── 工单系统
    │
    ├── MCP Client (Claude Code)
    │       └── 通过 HTTPS 调用 MCP Server 工具
    │
    ▼
Nginx (反向代理 + 静态文件)
    ├── /api/*          → 后端 API (Java, 端口 8080)
    ├── /mcp/*          → MCP Server (Node, 端口 3001)
    └── /               → 前端静态文件
    
后端 Java (Spring Boot)
    ├── 权限中间件 (读取权限配置，拦截请求)
    ├── 业务模块
    │   ├── RequirementService (需求 CRUD、状态流)
    │   ├── DevPlanService (进度树操作)
    │   ├── WikiService (知识库 CRUD、搜索)
    │   ├── TicketService (工单 CRUD)
    │   └── UserService (认证、API Key 管理)
    └── MongoDB 客户端

MCP Server (TypeScript)
    ├── Auth 模块 (API Key 验证 + 权限判定)
    ├── Tool Handlers (8个核心 Tool)
    └── API 客户端 (调用后端接口)

MongoDB (数据存储)
    ├── users, requirements, wiki_pages, tickets
    └── permission_rules (权限配置文档)
```

## 三、模块划分与接口清单

### 前端路由
- `/login` – 登录页
- `/requirements` – 需求列表
- `/requirements/new` – 创建需求
- `/requirements/:id` – 需求详情（含预览、文档视图、进度树 Tab、相关工单 Tab）
- `/tickets` – 工单看板
- `/tickets/new` – 创建工单
- `/tickets/:id` – 工单详情
- `/wiki` – 知识库首页（树状目录 + 内容）
- `/wiki/new` – 新建文档
- `/wiki/edit/:id` – 编辑文档
- `/settings` – 个人设置（API Key 管理）

### 后端 API 设计（`/api/v1`）

**认证**
- `POST /auth/login` – 登录，返回 JWT
- `POST /auth/register` – 注册（初期可由管理员手动添加）

**需求**
- `POST /requirements` – 创建
- `GET /requirements` – 列表（支持过滤状态）
- `GET /requirements/:id` – 详情
- `PUT /requirements/:id` – 更新
- `PATCH /requirements/:id/status` – 状态变更
- `GET /requirements/:id/source-html` – 获取源 HTML

**技术方案**
- `POST /tech-proposals` – 创建（关联需求）

**开发进度树**
- `POST /requirements/:id/dev-plan` – 创建/替换进度树
- `PATCH /requirements/:id/dev-plan/nodes/:nodeId` – 更新节点

**知识库**
- `POST /wiki/pages` – 创建
- `GET /wiki/pages` – 列表（支持按路径过滤）
- `GET /wiki/pages/:id` – 详情
- `PUT /wiki/pages/:id` – 更新
- `GET /wiki/search?q=&tags=&scope=` – 搜索

**工单**
- `POST /tickets` – 创建
- `GET /tickets` – 列表（筛选）
- `GET /tickets/:id` – 详情
- `PATCH /tickets/:id` – 更新（状态、指派等）
- `POST /tickets/:id/comments` – 添加评论

**用户 & API Key**
- `GET /users/me` – 当前用户信息
- `POST /users/me/api-keys` – 生成新 Key
- `DELETE /users/me/api-keys/:keyId` – 删除 Key

## 四、最终效果要求

1. **需求清晰化闭环**  
   - 老板上传模糊描述+HTML → 平台生成结构化需求草稿（初期可人工编辑，未来Agent辅助） → 可分享带沙箱预览的页面给老板确认 → 确认后自动通知开发。

2. **开发驾驭（Herness）**  
   - 需求确认后，开发可生成模块化进度树，每个节点独立追踪状态。  
   - 使用 Claude Code 的开发者在项目里连上 MCP，AI 自动读取需求、知识库，生成代码并同步进度到平台。  
   - 团队其他成员可在网页上实时看到开发到了哪个模块、产出了哪些 PR。

3. **知识沉淀与复用**  
   - 树状 Wiki 维护规范、环境配置、Agent Skills。  
   - 开发时 AI 可通过 MCP 实时搜索知识库，确保生成代码符合规范。  
   - 需求、进度、工单中均可关联知识库链接，减少重复解释。

4. **轻量工单流转**  
   - 测试/开发可快速提 Bug，关联到需求或进度节点，指派处理。  
   - 处理过程留痕，状态一目了然。

5. **安全与易用**  
   - 基于职能的权限控制，P0（老板）全通，其他人按职能隔离。  
   - Web 端统一登录，MCP 使用 API Key 一码接入，随时可撤销。

## 五、使用手册大纲（面向团队成员）

**老板/PM**
1. 如何提交新需求（上传 HTML 和描述）
2. 如何在“我的需求”里查看需求澄清状态
3. 如何通过沙箱预览确认需求效果
4. 如何查看需求的开发进度树

**开发人员**
1. 如何查看已确认的需求及详细文档
2. 如何生成并调整开发计划树
3. 如何获取 API Key 并配置 Claude Code MCP 连接
4. MCP 各工具的使用场景和示例
5. 如何更新进度节点、关联 PR 和文档
6. 如何查阅知识库和代码规范

**测试人员**
1. 如何创建 Bug 工单并关联需求
2. 如何跟踪指派给自己的工单
3. 如何通过需求进度树了解开发状态以制定测试计划
